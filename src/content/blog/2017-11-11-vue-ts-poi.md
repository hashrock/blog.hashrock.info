<h1>poi + TypeScript</h1>

<p>poiは設定より規約感のあるツールで、書捨てのプロジェクトにはwebpack直で使うよりpoiが楽だったりする。
.vueファイルへの対応も最初から組み込まれている。
デフォルトではbabelが使われる設定なのだけど、TypeScriptやる場合は、もう下記のことを順にやってけば良い。</p>

<p><iframe src="https://hatenablog-parts.com/embed?url=https%3A%2F%2Fgithub.com%2Fegoist%2Fpoi%2Ftree%2Fmaster%2Fpackages%2Fpoi-preset-typescript" title="egoist/poi" class="embed-card embed-webcard" scrolling="no" frameborder="0" style="display: block; width: 100%; height: 155px; max-width: 500px; margin: 10px 0px;"></iframe><cite class="hatena-citation"><a href="https://github.com/egoist/poi/tree/master/packages/poi-preset-typescript">github.com</a></cite></p>

<p>で、それなりにボイラープレートも必要だったので、自分なりにやった手順を書いておく。</p>

<p>まずはセットアップ。</p>

<pre class="code" data-lang="" data-unlink>mkdir my-project
cd $_
yarn init
yarn add poi typescript poi-preset-typescript --dev</pre>


<p>そして、下記のファイルを作る。</p>

<p>index.ts</p>

<pre class="code" data-lang="" data-unlink>import Vue from &#39;vue&#39;
import App from &#34;./App.vue&#34;

new Vue({
  el: &#39;#app&#39;,
  render: h =&gt; h(App)
})</pre>


<p>tsconfig.json</p>

<pre class="code" data-lang="" data-unlink>{
  &#34;compilerOptions&#34;: {
    &#34;target&#34;: &#34;es5&#34;,
    &#34;strict&#34;: true,
    &#34;module&#34;: &#34;es2015&#34;,
    &#34;moduleResolution&#34;: &#34;node&#34;,
    &#34;experimentalDecorators&#34;: true
  }
}</pre>


<p>App.vue</p>

<pre class="code" data-lang="" data-unlink>&lt;template&gt;
  &lt;div id=&#34;main&#34;&gt;
    &lt;h1&gt;Hello World!&lt;/h1&gt;
    &lt;input v-model=&#34;text&#34;&gt;
    &lt;p&gt;input: {{text}}&lt;/p&gt;
  &lt;/div&gt;
&lt;/template&gt;
&lt;script lang=&#34;ts&#34;&gt;
import Vue from &#34;vue&#34;;
import Component from &#34;vue-class-component&#34;;

@Component({})
export default class App extends Vue {
  text: string = &#34;Hello There&#34;;
  mounted() {}
}
&lt;/script&gt;
</pre>


<p>poi.config.js</p>

<pre class="code" data-lang="" data-unlink>module.exports = {
  presets: [
    require(&#39;poi-preset-typescript&#39;)(/* options */)
  ],
  &#34;homepage&#34;: &#34;./&#34;
}</pre>


<p>sfc.d.ts</p>

<pre class="code" data-lang="" data-unlink>declare module &#34;*.vue&#34; {
  import Vue from &#39;vue&#39;
  export default Vue
}</pre>


<p>その後、package.json内に下記を追記。</p>

<pre class="code" data-lang="" data-unlink>  &#34;scripts&#34;: {
    &#34;dev&#34;: &#34;poi index.ts&#34;,
    &#34;build&#34;: &#34;poi build index.ts&#34;
  }
</pre>


<p>もし、ビルド後のプロジェクトをGitHub Pagesにデプロイしたい場合は…</p>

<pre class="code" data-lang="" data-unlink>yarn add gh-pages
</pre>


<p>しつつ、package.jsonに以下も追加</p>

<pre class="code" data-lang="" data-unlink>  &#34;scripts&#34;: {
    &#34;deploy&#34;: &#34;gh-pages -d dist&#34;
  }</pre>


<p>という感じで、意外とボイラープレート作らなきゃいけなくて、
まだ楽とは言い切れない感じではある。</p>

<p>ただこれさえ済ませてしまえば、Vueファイルの中でもTypeScriptが使えてバイブス上がる感じになる。</p>

<p><span itemscope itemtype="http://schema.org/Photograph"><img src="https://cdn-ak.f.st-hatena.com/images/fotolife/h/hashrock/20171111/20171111031211.png" alt="f:id:hashrock:20171111031211p:plain" title="f:id:hashrock:20171111031211p:plain" class="hatena-fotolife" itemprop="image"></span></p>

<p>という風に、補完もバッチリ効く。ただ、LSP実装もなかなか大変なようで、リネームやリファクタリングには未対応の様子だった。<a href="https://github.com/vuejs/vetur/issues/188">veturのロードマップ</a></p>

<p>VSCodeのリファクタ機能をフルに使いたい場合は、ロジックはなるべくモデルに切り出して、普通のTSにしてからリファクタするのが良さそう。</p>

<h1>気持ち</h1>

<p>自分がコード書くときはとにかく設計も何もなく適当にぐちゃぐちゃ書き進めることが大半で、しかも意識が朦朧としていることが多かったりする。JSは集中してないと書けないので大変難しい。</p>

<p>自分がアホだからこそ役立つのがTypeScriptなんだけど、アホにとってtsconfig.jsonやwebpack.config.jsはパズルじみていて、書くのが難しいという問題がある。もうちょっと楽になったらいいなぁとは思う。</p>
