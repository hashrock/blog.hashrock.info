<p>例えばnpm initは最初にこんな感じのダイアログが出ます。</p>

<pre class="code" data-lang="" data-unlink>name: (test) test
version: (1.0.0)
description:
entry point: (index.js)
test command:
git repository:
keywords:
author:</pre>


<p>yeomanとかも一緒です。</p>

<p>これ、基本的にEnterを連打する類のものではありますが、
基本的に入力したら戻れません。
（とかいって戻れたら赤っ恥ですが）</p>

<p>なぜこうではいけないのでしょうか。</p>

<p><img src="https://cloud.githubusercontent.com/assets/3132889/12170454/e3fb2bc4-b584-11e5-92f8-637e4d13a83b.gif" alt="cli" /></p>

<p>これなら、こんな感じの利点がありますね。</p>

<ul>
<li>好きな順序で入力できる</li>
<li>デフォルト値でよい項目は、エンターで確定するのではなく、単に無視すればよい</li>
<li>残項目がいくつあるのか分かる</li>
</ul>


<p>というわけで作ってみました（ライブラリ化はしてません）。</p>

<p><a href="https://github.com/hashrock/idea-friendly-dialog-cli">hashrock/idea-friendly-dialog-cli &middot; GitHub</a></p>

<p>もちろん、既存のものの方がシンプルな実装なはずで、より多くの環境で動きそうですし、
npm本体やyeoman本体にこれを送りつける自信はありません。</p>

<p>こんなア<a class="keyword" href="http://d.hatena.ne.jp/keyword/%A5%A4%A5%C7%A5%A2">イデア</a>もあるということで。</p>

<p>追記：そういえば、カーソルキーの使いにくい環境もありそうですね。</p>
