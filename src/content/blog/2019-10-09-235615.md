<h1>DenoでJSXが実行できるようになったよ</h1>

<p><blockquote class="twitter-tweet" data-lang="HASH(0x563c648d8920)"><p lang="ja" dir="ltr">react勢みて、denoで<a class="keyword" href="http://d.hatena.ne.jp/keyword/tsx">tsx</a>直接実行できる <a href="https://t.co/2FRMRK3Vda">pic.twitter.com/2FRMRK3Vda</a></p>&mdash; hashrock (@hashedrock) <a href="https://twitter.com/hashedrock/status/1181496564155445248?ref_src=twsrc%5Etfw">October 8, 2019</a></blockquote><script async src="https://platform.twitter.com/widgets.js" charset="utf-8"></script></p>

<p>なりました。</p>

<p><blockquote class="twitter-tweet" data-lang="HASH(0x563c648d8920)"><p lang="ja" dir="ltr">かつてここまでコピペで動くwebサーバーがあっただろうか <a href="https://twitter.com/hashtag/denoland?src=hash&amp;ref_src=twsrc%5Etfw">#denoland</a><br> <a href="https://t.co/nSAaLC1RIi">pic.twitter.com/nSAaLC1RIi</a></p>&mdash; keroxp (@keroxp) <a href="https://twitter.com/keroxp/status/1181554197973495808?ref_src=twsrc%5Etfw">October 8, 2019</a></blockquote><script async src="https://platform.twitter.com/widgets.js" charset="utf-8"></script></p>

<p>このかっこいいサーバを見てほしいんですが、renderToStringの中身が普通にJSXです。</p>

<p>「かつてここまでコピペで動くwebサーバーがあっただろうか」とありますが、denoはまじでこの１ファイルだけで動きます。</p>

<p>文字列でもテンプレート<a class="keyword" href="http://d.hatena.ne.jp/keyword/%A5%EA%A5%C6%A5%E9%A5%EB">リテラル</a>でもなく、普通にDOMを<a class="keyword" href="http://d.hatena.ne.jp/keyword/%A5%BD%A1%BC%A5%B9%A5%B3%A1%BC%A5%C9">ソースコード</a>中に書けるし、無論JSX<a class="keyword" href="http://d.hatena.ne.jp/keyword/%A5%B3%A5%F3%A5%DD%A1%BC%A5%CD%A5%F3%A5%C8">コンポーネント</a>を作ってimportしてくるみたいなのもできるし、それを普通にレスポンスとして返せば<a class="keyword" href="http://d.hatena.ne.jp/keyword/SSR">SSR</a>になるわけです。なおimportの目先にあるjspm.ioというのは、npmパッケージをES Modulesに変換してくれる<a class="keyword" href="http://d.hatena.ne.jp/keyword/CDN">CDN</a>です。</p>

<p>webpack.config.jsもtsconfigも必要ないし、npm installももちろん不要です。ただただ実行時にimportからリンクされているファイルがキャッシュされて、<a class="keyword" href="http://d.hatena.ne.jp/keyword/TSX">TSX</a>のトランスパイルが走ってサーバが立ちます。</p>

<p>おまけにセキュアです。ファイルや<a class="keyword" href="http://d.hatena.ne.jp/keyword/%B4%C4%B6%AD%CA%D1%BF%F4">環境変数</a>やネットワークアクセスなども許可制になっているので、依存ライブラリがおもむろに<a class="keyword" href="http://d.hatena.ne.jp/keyword/%B4%C4%B6%AD%CA%D1%BF%F4">環境変数</a>盗んで怪しいサーバに送信していたみたいなリスクが低減されています。</p>

<p>スマートですごくないですか！？みたいな話をいろんなところでめっちゃしているんですが、まぁうまく伝わらないというか、実用上の利点や採用例が提示できないとなかなか広まらないのかなーという感じです。</p>

<p>聞くところによると、<a href="https://twitter.com/deno_community/status/1181871786981388289">Deno経験者募集</a>的なものがついに出たらしく、<b>世界初の「Denoで食ってる人」</b>が<a class="keyword" href="http://d.hatena.ne.jp/keyword/%C7%FA%C3%C2">爆誕</a>するのを期待したいところです。</p>

<p>JSX入ったのはNode.jsにはない面白い特徴なんで、Reactユーザにこの週末あたりにおもちゃにしてもらって、エクストリームな活用方法を提示してくれたらDenoももう一盛り上がりする気がします。</p>

<p>ちなみにJSXサポートを入れたのはおなじみkeroxpさんです。スレ内に「JSXなんて入れやがって」みたいに怒っている人がいますがこれは誤解で、Ryanが説明している通りDeno内にJSX実装は入っていません。TypeScriptを昔から触っている人はピンとくると思いますが、<a class="keyword" href="http://d.hatena.ne.jp/keyword/tsc">tsc</a>側にもとからJSXサポートが入っています。Deno側でファイルタイプで制約がかかっていたのが外れた感じだと思います。</p>

<p><a href="https://github.com/denoland/deno/pull/3038">feat: JSX Support by keroxp &middot; Pull Request #3038 &middot; denoland/deno &middot; GitHub</a></p>

<p>個人的には、TypeScriptは言語なのにReactのテンプレートであるJSXのサポートが入っているのはなにか違うんじゃないかなと思ってたんですが、回り回ってDenoに強力なテンプレートエンジンとしてのJSXが降ってきた感じでややシュールです。しかし入ってみると実際便利そうですね。型も付くし間違ったタグを書けばエラーになるしインジェクションも防げますよね。Mustache使うとかに比べると相当堅牢になるんじゃないでしょうか。</p>
